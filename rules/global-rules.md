# 全局规则

## 教程目标

1. 使读者能够独立设计、开发并部署一个支持多登录方式（Google登录、微信扫码登录、GitHub登录以及邮箱 Magic Link 登录）、具备单点登录能力的集中式认证服务。
2. 让读者不仅会用，还能深入理解认证原理和最佳实践。

## 教程受众

- 零认证知识基础
- 有 TypeScript 基础
- 希望深入理解原理，不只是会用

## 技术栈

- 后端
  - Nest.js 11
  - TypeScript 5
  - Drizzle ORM
  - Zod 4
  - PostgreSQL 18
  - Redis 8

- 前端

## 教程写作规范

### 内容风格

详见[内容风格要求](./content-style-rules.md)，包括图表、标题、列表、code block等的具体格式要求。

### 结构模板

每篇教程必须严格按照以下顺序编写：

1. 本篇导读（核心目标、重点难点、预计阅读时间）
2. 前置知识（仅列出本篇必需的前置内容，可链接到已完成的篇章）
3. 核心概念讲解（原理性内容，用通俗易懂的语言+类比解释）
4. 实操步骤（分步骤，每步配代码示例+注释）
5. 常见问题与解决方案
6. 本篇小结

### 内容深度要求

- 每个概念必须解释"为什么"和"怎么做"
- 对比不同方案的优缺点
- 包含实际生产环境的最佳实践
- 包含安全考虑和常见陷阱

### 质量标准

- 字数：每章2万字以上
- 风格：通俗易懂，避免过多术语，必要时解释术语
- 图表：概念可借助图表说明，图表必须清晰、易懂，且与文本内容紧密相关
