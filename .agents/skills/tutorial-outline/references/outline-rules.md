# 大纲规则

大纲需包含一下内容：

1. 模块划分：将教程内容分成若干模块，每个模块聚焦一个主题
2. 教程文章列表：每个模块下列出教程文章
3. 每篇文章包含以下信息：
   - 序号：模块内的顺序编号
   - 标题。如果教程已完成，则标题形式如：`[认证 vs 授权](./docs/auth-vs-authz.md)`。如果教程未完成，则标题形式如：`认证 vs 授权`。
   - 教程完成度：已完成的文章用 ✅ 标记，未完成的文章用 ⬜ 标记
   - 内容关键字：3-5个关键词，概括文章的核心内容，便于读者快速了解文章主题
   - 内容摘要：300字左右的文章摘要，摘要需要以教程的内容为基础，不能凭空编造。摘要的目的是让读者在浏览大纲时，能够快速了解每篇文章的核心内容和价值，从而决定是否深入阅读。

格式如下：

```markdown
## 模块一：Foo

1. [认证 vs 授权](./docs/auth-vs-authz.md)
   - 完成度：✅
   - 内容关键字：<这里是关键词列表，3-5个关键词，用逗号分隔>
   - 内容摘要：<这里是300字左右的文章摘要>
2. [SSO 与集中式认证服务](./docs/bar-tutorial-file.md)
```
